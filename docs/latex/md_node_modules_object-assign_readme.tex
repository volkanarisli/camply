\subsection*{Use the built-\/in}

Node.\+js 4 and up, as well as every evergreen browser (Chrome, Edge, Firefox, Opera, Safari), support {\ttfamily Object.\+assign()} \+:tada\+:. If you target only those environments, then by all means, use {\ttfamily Object.\+assign()} instead of this package.

\subsection*{Usage}


\begin{DoxyCode}
const objectAssign = require('object-assign');

objectAssign(\{foo: 0\}, \{bar: 1\});
//=> \{foo: 0, bar: 1\}

// multiple sources
objectAssign(\{foo: 0\}, \{bar: 1\}, \{baz: 2\});
//=> \{foo: 0, bar: 1, baz: 2\}

// overwrites equal keys
objectAssign(\{foo: 0\}, \{foo: 1\}, \{foo: 2\});
//=> \{foo: 2\}

// ignores null and undefined sources
objectAssign(\{foo: 0\}, null, \{bar: 1\}, undefined);
//=> \{foo: 0, bar: 1\}
\end{DoxyCode}


\subsection*{A\+PI}

\subsubsection*{object\+Assign(target, \mbox{[}source, ...\mbox{]})}

Assigns enumerable own properties of {\ttfamily source} objects to the {\ttfamily target} object and returns the {\ttfamily target} object. Additional {\ttfamily source} objects will overwrite previous ones.
